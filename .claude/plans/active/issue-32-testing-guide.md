# Issue #32 Testing Guide

## 测试概览

本指南涵盖 Issue #32 (UX Polish - Backend Integration) 的所有测试要求。

## 4.1 文件操作测试

### 删除文件测试

**测试目标**: 验证文件删除功能正确清理数据库和物理文件

**手动测试步骤**:
1. 导入一个测试文件到 U-Safe
2. 加密该文件
3. 在 UI 中选择文件并点击删除
4. 验证数据库记录已删除：
   - 检查 `files` 表中是否还有该记录
   - 检查 `file_tags` 表中关联记录是否级联删除
   - 检查 `encryption_meta` 表中关联记录是否级联删除
5. 验证物理文件已删除：
   - 原始文件路径不存在
   - 加密文件路径不存在

**预期结果**: ✅ 所有数据库记录和物理文件都被正确清理

**单元测试**: `src-tauri/src/commands/file_operations.rs::tests::test_delete_file_success`

---

### 重命名文件测试

**测试目标**: 验证文件重命名功能正确更新数据库

**手动测试步骤**:
1. 导入一个测试文件（如 `test.txt`）
2. 在 UI 中双击文件名进入编辑模式
3. 输入新文件名（如 `renamed.txt`）
4. 按 Enter 确认
5. 验证数据库 `files` 表中 `original_name` 字段已更新
6. 验证 `updated_at` 时间戳已更新

**边界测试**:
- ❌ 空文件名 → 应显示错误提示
- ❌ 超过 255 字符 → 应显示错误提示
- ❌ 包含非法字符 (`/`, `\`, `:`, `*`, `?`, `<`, `>`, `|`, `"`) → 应显示错误提示

**预期结果**: ✅ 文件名正确更新，非法输入被拒绝

**单元测试**: `src-tauri/src/commands/file_operations.rs::tests::test_rename_file_validation`

---

## 4.2 进度事件测试

### 加密进度测试

**测试目标**: 验证加密过程中进度事件准确性

**手动测试步骤**:
1. 准备一个大文件（>10MB）
2. 开始加密
3. 观察 UI 中的 ProgressBar 组件
4. 验证进度百分比从 0% 递增到 100%
5. 验证进度条平滑更新（无跳跃）

**预期结果**: ✅ 进度准确反映加密进度

**验证点**:
- 事件名称: `encryption-progress`
- 数据结构: `{ bytes_processed, total_bytes, percentage }`
- 百分比计算: `(bytes_processed / total_bytes) * 100`

---

### 解密进度测试

**测试目标**: 验证解密过程中进度事件准确性

**手动测试步骤**:
1. 准备一个已加密的大文件（>10MB）
2. 开始解密
3. 观察 UI 中的 ProgressBar 组件
4. 验证进度百分比从 0% 递增到 100%

**预期结果**: ✅ 进度准确反映解密进度

**验证点**:
- 事件名称: `decryption-progress`
- 数据结构: `{ bytes_processed, total_bytes, percentage }`

---

## 4.3 错误恢复测试

### 加密中断恢复测试

**测试目标**: 验证加密过程中断后的错误处理

**手动测试步骤**:
1. 开始加密一个大文件
2. 在加密过程中强制关闭应用
3. 重新启动应用
4. 检查数据库状态
5. 检查物理文件状态

**预期结果**: ✅ 没有损坏的数据，可以重新加密

---

### 磁盘满测试

**测试目标**: 验证磁盘空间不足时的错误处理

**手动测试步骤**:
1. 准备一个 U 盘，填充至接近满（留少于文件大小的空间）
2. 尝试加密一个大文件
3. 观察错误提示

**预期结果**: ✅ 显示清晰的错误信息（"磁盘空间不足"）

---

## 4.4 无障碍测试

### 键盘导航测试

**测试目标**: 验证所有功能可通过键盘访问

**手动测试步骤**:
1. 使用 Tab 键导航到文件列表
2. 使用方向键选择文件
3. 按 Enter 打开文件
4. 按 Delete 删除文件
5. 按 F2 重命名文件
6. 按 Escape 取消操作

**预期结果**: ✅ 所有操作都可以通过键盘完成

---

### 屏幕阅读器测试

**测试目标**: 验证屏幕阅读器兼容性

**测试工具**:
- Windows: NVDA (https://www.nvaccess.org/)
- macOS: VoiceOver (内置)

**手动测试步骤**:
1. 启动屏幕阅读器
2. 导航到文件列表
3. 验证文件名、状态（已加密/未加密）被正确朗读
4. 验证按钮有正确的 ARIA 标签
5. 验证进度条有正确的 `aria-valuenow` 属性

**预期结果**: ✅ 所有 UI 元素都有适当的 ARIA 属性

**参考**: `.claude/rules/frontend/design-system.md` - Accessibility 部分

---

### 颜色对比度测试

**测试目标**: 验证 WCAG 2.1 AA 颜色对比度标准

**测试工具**: Axe DevTools (浏览器扩展)

**手动测试步骤**:
1. 在浏览器中打开应用
2. 安装 Axe DevTools
3. 运行自动检测
4. 检查报告中的颜色对比度问题

**预期结果**: ✅ 无颜色对比度警告或错误

**最低标准**:
- 正常文字: 4.5:1
- 大号文字 (18pt+): 3:1
