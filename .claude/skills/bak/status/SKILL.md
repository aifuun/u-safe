# Status - Combined Project Overview

Show comprehensive project status with dual output: terminal text + HTML report.

## Task

When invoked, generate:
1. **Terminal Output**: Quick overview (git, plans, issues, metrics)
2. **HTML Report**: Full 5-tab report (Overview, Status, Architecture, Quality, History)

Both outputs are generated by default unless flags override.

## Flags

- `--text-only`: Skip HTML generation
- `--html-only`: Skip terminal output, only generate HTML
- `--no-open`: Don't auto-open HTML in browser (default: open)

## Data Collection

### Git Status (LIVE)
```bash
git rev-parse --abbrev-ref HEAD                  # Branch
git rev-parse --short HEAD                       # Commit
git log -1 --format="%s"                        # Message
git diff --cached --name-only | wc -l           # Staged
git diff --name-only | wc -l                    # Unstaged
git ls-files --others --exclude-standard | wc -l # Untracked
git log --oneline -10 --format="%h|%s|%ar"      # Recent commits
```

### Framework Analysis
```bash
# Parse .framework-install (JSON or Markdown format)
cat .framework-install

# Count pillars
find .prot/pillars -type d -name "pillar-*" | wc -l

# Count rules
find .claude/rules -type f -name "*.md" | wc -l

# Count commands
find .claude/commands -type f -name "*.md" | wc -l
```

### Active Work
```bash
# Active plans
ls -1 .claude/plans/active/*.md

# Open issues (if gh CLI available)
gh issue list --state open --limit 10 --json number,title,state
```

### Pattern Detection
```bash
# Nominal typing
grep -r "unique symbol" src/

# Airlock validation
grep -r "airlock" src/

# Saga pattern
grep -r "Saga" src/

# Headless UI
find src -type d -name "headless"

# Test coverage
find tests -type f \( -name "*.test.*" -o -name "*.spec.*" \)
```

### Health Score Calculation (0-100)
- Framework installed: +20
- Has tests: +20
- Uses nominal types: +15
- Uses airlock: +15
- Has active plans: +10
- Clean git status: +10
- Documentation exists: +10

## HTML Output (5 Tabs)

### Tab 1: Overview
- Quick stats (pillars, rules, commands, health score)
- Profile information
- Generated timestamp

### Tab 2: Status
- Git information (branch, commit, message)
- Working directory (staged, unstaged, untracked)
- Active plans
- Open issues

### Tab 3: Architecture
- Framework profile
- Installed pillars (grid display with descriptions)
- Rules and commands count

### Tab 4: Code Quality
- Health score
- Detected patterns
- Strengths
- Observations
- Prioritized recommendations

### Tab 5: History
- Recent 10 commits
- Commit details (hash, message, date)

## Implementation Steps

1. **Parse flags** (`--text-only`, `--html-only`, `--no-open`)
2. **Collect all data** (git, framework, patterns, health)
3. **Generate terminal output** (unless `--html-only`)
4. **Generate HTML report** (unless `--text-only`)
5. **Write HTML to `docs/reports/`** with timestamp
6. **Auto-open browser** (unless `--no-open`)
7. **Display success message** with file path

## Error Handling

- Git not available → Use placeholder values
- Framework not installed → Show "Not installed"
- GitHub CLI missing → Skip issues section
- Write permission error → Clear error message

## Example Usage

```bash
# Default: Both text + HTML
/status

# Terminal only (fast)
/status --text-only

# HTML only (shareable report)
/status --html-only

# Generate HTML but don't open
/status --no-open
```
