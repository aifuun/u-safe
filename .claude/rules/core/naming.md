# Naming Rules

> **Golden Rule**: Single source of truth in `product.config.json` Ôćĺ `technicalName` field.

See full decision: [ADR-022](../docs/architecture/ADR/022-single-source-product-config.md)

---

## Quick Reference

### Three-Tier Naming Separation

| Tier | Source | Changes | Examples |
|------|--------|---------|----------|
| **User-Visible** | `productName` in config | Frequently | "Recie", "ŃâČŃéĚŃéĘ" |
| **Code Structure** | Hardcoded (neutral) | Never | `MainStack`, `app_lib` |
| **Runtime Resources** | `technicalName` in config | Rarely | `recie-*`, `recie.db` |

## Rules by Context

### ÔŁî What NOT to Do

```typescript
// WRONG - Hardcoded product name in code
export class RecieStack extends cdk.Stack { }
const dbName = "recie.db";
const bucket = `recie-images-${env}`;

// WRONG - Brand-coupled class names
export class YorutsukeService { }
pub fn yorutsuke_init() { }
```

### Ôťů What TO Do

```typescript
// RIGHT - Neutral code structure
export class MainStack extends cdk.Stack { }
export class AdminStack extends cdk.Stack { }

// RIGHT - From generated config
import { DB_PRODUCTION, RESOURCE_PREFIX } from './generated/config';
const dbName = DB_PRODUCTION;  // "recie.db"

// RIGHT - Use helper functions
const bucket = getS3BucketName("images", "us", env, account);
// Ôćĺ "recie-images-us-dev-696249060859"
```

## Generated Files (Auto-Updated)

Three files are automatically generated by `scripts/sync-product-config.js`:

1. **`app/src/generated/config.ts`** (Frontend)
   ```typescript
   export const DB_PRODUCTION = 'sqlite:recie.db';
   export const STORAGE_PREFIX = 'recie';
   export const LOG_DIRECTORY = '.recie';
   ```

2. **`infra/lib/generated/config.ts`** (Infrastructure)
   ```typescript
   export const RESOURCE_PREFIX = 'recie';
   export function getS3BucketName(type, region, env, account) {
     return `${RESOURCE_PREFIX}-${type}-${region}-${env}-${account}`;
   }
   ```

3. **`app/src-tauri/src/generated/config.rs`** (Rust)
   ```rust
   pub const DB_PRODUCTION: &str = "recie.db";
   pub const LOG_DIRECTORY: &str = ".recie";
   ```

## Rules by Layer

### Frontend (React/TypeScript)

| Need | Rule |
|------|------|
| Database name | `import { DB_PRODUCTION } from './generated/config'` |
| Storage key | `import { STORAGE_PREFIX } from './generated/config'` |
| Display name | `import { PRODUCT_NAME } from './generated/config'` |
| Log directory | `import { LOG_DIRECTORY } from './generated/config'` |

### Infrastructure (CDK)

| Need | Rule |
|------|------|
| S3 bucket | `getS3BucketName(type, region, env, account)` |
| DynamoDB table | `getDynamoTableName(tableName, region, env)` |
| Lambda function | `getLambdaFunctionName(functionName, region, env)` |
| Cognito pool | `getCognitoPoolName(poolType, env)` |
| Lambda Layer | `getLayerName(layerName, env)` |
| Stack name | `getStackName(env, 'main' \| 'admin')` |

### Backend (Rust)

| Need | Rule |
|------|------|
| Database name | `use crate::generated::config::DB_PRODUCTION;` |
| Log directory | `use crate::generated::config::LOG_DIRECTORY;` |
| Product name | `use crate::generated::config::PRODUCT_NAME;` |

## Common Scenarios

### Scenario 1: Adding New AWS Resource

```typescript
// infra/lib/main-stack.ts
import { getLambdaFunctionName } from './generated/config';

// Ôťů CORRECT - Uses helper
const newLambda = new lambda.Function(this, "NewFeature", {
  functionName: getLambdaFunctionName("new-feature", "us", env),
  // Ôćĺ "recie-new-feature-us-dev"
});

// ÔŁî WRONG - Hardcoded
functionName: `recie-new-feature-us-${env}`
```

### Scenario 2: Adding New Database File

```typescript
// app/src/00_kernel/storage/db.ts
import { DB_PRODUCTION } from '../../generated/config';

// Ôťů CORRECT - From config
const db = await Database.load(DB_PRODUCTION);

// ÔŁî WRONG - Hardcoded
const db = await Database.load('sqlite:recie.db');
```

### Scenario 3: Adding New Storage Key

```typescript
// app/src/01_domains/*/storage.ts
import { STORAGE_PREFIX } from '../../generated/config';

// Ôťů CORRECT - Derived from prefix
const FEATURE_KEY = `${STORAGE_PREFIX}:feature`;  // "recie:feature"

// ÔŁî WRONG - Hardcoded
const FEATURE_KEY = 'recie:feature';
```

### Scenario 4: Creating New Stack Class

```typescript
// infra/lib/new-stack.ts

// Ôťů CORRECT - Neutral name
export class ProcessingStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);
    // Resource names from config helpers
  }
}

// ÔŁî WRONG - Brand-coupled
export class RecieProcessingStack extends cdk.Stack { }
```

## Checklist Before Committing

- [ ] No hardcoded product names in code
- [ ] Stack classes use neutral names (`MainStack`, not `RecieStack`)
- [ ] AWS resources use helper functions from generated config
- [ ] Database names imported from generated config
- [ ] Storage keys use `STORAGE_PREFIX` constant
- [ ] No brand names in class/function/variable names

## Rationale

### Why Neutral Code Structure?

Industry examples of brand-agnostic code:
- **React**: Code unchanged when Facebook Ôćĺ Meta
- **AWS CDK**: Uses `Stack`, not `AmazonStack`
- **Kubernetes**: Uses `Cluster`, not `GoogleCluster`

**Principle**: Code structure describes function, not brand identity.

### Why Single Source?

**Before**: 50+ locations to update manually
- High risk of missing some
- Drift and inconsistency
- Manual search-and-replace errors

**After**: One field change updates everything
- Zero drift
- Type-safe
- Auto-generation ensures consistency
