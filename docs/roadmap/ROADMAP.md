## MVP - v1.0 (2026 Q1)

**🎯 主题**: 核心价值验证 — 证明"加密 + 标签"模式可行

**📅 目标日期**: 2026-05-15 (MVP 完整版)
- **Alpha**: 2026-03-31 (M2 加密引擎演示)
- **Beta**: 2026-04-18 (M4 核心功能完整)

**👥 目标用户**: 10-20 名内测用户（技术背景）

### 核心特性

#### ✅ P0 必须完成

| 功能 | 描述 | 对应 Issue |
|------|------|-----------|
| 密码管理 | 主密码设置/验证 (Argon2id KDF) | #1 |
| 文件加密 | AES-256-GCM 分块加密 (64KB) | #2 |
| 文件解密 | 临时解密 + 自动清理 | #3 |
| 物理视图 | 树状文件浏览 + 基础操作 | #4 |
| 标签系统 | 层级标签 CRUD + 文件打标签 | #5 |
| 标签视图 | 按标签分组展示文件 | #6 |
| 视图切换 | 物理 ↔ 标签无缝切换 | #7 |

#### 🔄 P1 尽力完成

| 功能 | 描述 | 对应 Issue |
|------|------|-----------|
| 进度显示 | 加密/解密进度条 | #8 |
| 文件操作 | 重命名、删除、新增 | #9 |
| 标签编辑 | 重命名、删除、层级调整 | #10 |

### 成功标准

- [ ] 能加密/解密 100MB 文件，性能 > 50 MB/s
- [ ] 能管理 100 个标签，层级 ≤ 5 层
- [ ] Windows 10/11 + macOS (Intel/Apple Silicon) 正常运行
- [ ] 冷启动时间 < 2 秒 (SSD U盘)
- [ ] 应用体积 < 10 MB
- [ ] 零 P0 级别 Bug

### 不包含

- ❌ 多语言支持（仅中文）
- ❌ 批量操作
- ❌ 文件搜索
- ❌ 云同步

### 里程碑

详见：[MVP_Definition.md](./MVP_Definition.md)

**实施计划**: [MVP_v1.0_Implementation_Plan.md](./MVP_v1.0_Implementation_Plan.md)

**Status**: 🟡 In Progress (M1 项目骨架)

## v1.1 (2026 Q2)

**🎯 主题**: 易用性提升 — 从"能用"到"好用"

**📅 预计日期**: 2026-06-30

**👥 目标用户**: 100-200 名公开测试用户

### 新增特性

| 功能 | 描述 | 优先级 | 预估工作量 |
|------|------|--------|-----------|
| 🌐 多语言支持 | 中文/英文/日文界面 | P0 | 3 周 |
| 📦 批量操作 | 批量加密/解密/打标签 | P0 | 2 周 |
| 🔍 高级搜索 | FTS5 全文搜索（文件名/标签） | P1 | 2 周 |
| 🔔 自动更新 | Tauri Updater 集成 | P1 | 1 周 |
| 🎨 系统托盘 | 最小化到托盘 + 快速操作 | P2 | 1 周 |
| ⌨️ 快捷键 | 常用操作快捷键支持 | P2 | 1 周 |

### 改进特性

- 标签拖拽排序
- 文件预览（图片缩略图）
- 更友好的错误提示
- 性能优化（大文件处理 > 500MB）

### 成功标准

- [ ] 支持 3 种语言（中/英/日）
- [ ] 批量操作 100 个文件 < 30 秒
- [ ] 搜索响应时间 < 100ms (1000 个文件)
- [ ] 自动更新成功率 > 95%
- [ ] 用户满意度 > 4.0/5.0

### 风险

- 多语言翻译质量（缓解：专业翻译服务）
- 批量操作内存占用（缓解：流式处理）

**Status**: 🔵 Planned

## v1.2 (2026 Q3)

**🎯 主题**: 企业级特性 — 面向专业用户

**📅 预计日期**: 2026-09-30

**👥 目标用户**: 企业用户、高级个人用户

### 新增特性

| 功能 | 描述 | 优先级 | 预估工作量 |
|------|------|--------|-----------|
| 🔐 密钥备份/恢复 | 安全的密钥导出/导入机制 | P0 | 3 周 |
| 📊 使用统计 | 加密文件数、容量、操作历史 | P1 | 2 周 |
| 🔄 文件版本历史 | 保留最近 N 个版本 | P1 | 3 周 |
| 🚨 U盘拔出检测 | 检测 U 盘移除，防止数据损坏 | P1 | 2 周 |
| 📝 操作日志 | 审计日志（谁、何时、做了什么） | P2 | 1 周 |

### 改进特性

- 加密算法升级选项（支持 ChaCha20-Poly1305）
- 大文件处理优化（> 1GB）
- 数据库备份/恢复功能

### 成功标准

- [ ] 密钥恢复成功率 100%（测试 1000 次）
- [ ] 支持文件版本历史（最多 10 个版本）
- [ ] U盘拔出检测准确率 > 99%
- [ ] 大文件加密性能 > 100 MB/s (> 1GB)

### 不确定性

- 文件版本历史的存储开销（可能延后到 v2.0）
- 密钥备份的安全模型（需要安全审计）

**Status**: 🔵 Planned

## v2.0 (2026 Q4)

**🎯 主题**: 云服务集成 — 突破"离线限制"

**📅 预计日期**: 2026-12-31

**👥 目标用户**: 多设备用户、团队协作用户

### 重大变更

⚠️ **架构演进**: 从"离线优先"到"混合模式"（本地 + 云端）

### 新增特性

| 功能 | 描述 | 优先级 | 预估工作量 |
|------|------|--------|-----------|
| ☁️ 云同步 | E2E 加密的云端备份 | P0 | 6 周 |
| 🔗 文件分享 | 生成限时加密分享链接 | P0 | 4 周 |
| 👥 多设备协同 | 同一账号多设备同步 | P0 | 4 周 |
| 🌍 Web 版本 | 浏览器查看模式（只读） | P1 | 4 周 |
| 🏢 团队空间 | 共享标签库和文件 | P2 | 6 周 |

### 技术架构变更

- **新增后端服务**: Rust + AWS (Lambda + DynamoDB + S3)
- **新增身份认证**: Cognito 用户管理
- **新增 API Gateway**: RESTful API for sync

### 成功标准

- [ ] 云同步延迟 < 5 秒（100MB 文件）
- [ ] 分享链接访问成功率 > 99.9%
- [ ] 多设备冲突解决准确率 100%
- [ ] Web 版本支持主流浏览器（Chrome/Safari/Firefox）

### 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 云服务成本高 | 运营负担 | 引入订阅模式（免费版 + Pro 版） |
| 网络依赖 | 离线场景受限 | 保留完整的离线模式 |
| 隐私顾虑 | 用户信任 | E2E 加密 + 开源审计 + 隐私协议 |
| 服务器运维 | 技术复杂度 | 使用 Serverless（AWS Lambda） |

### 兼容性

- **向后兼容**: v2.0 可以打开 v1.x 创建的加密文件
- **不兼容**: 云同步功能需要账号注册（可选）

**Status**: 🔵 Planned

## 决策记录

记录重要的路线图决策和变更历史。

### 2026-03-14: MVP 范围确定

**决策**: MVP 不包含多语言、批量操作、云同步
**理由**: 聚焦核心价值验证，控制开发周期
**影响**: 延后到 v1.1 和 v2.0

### 2026-03-14: 云服务延后到 v2.0

**决策**: 云同步和文件分享从 v1.2 延后到 v2.0
**理由**: 需要完整的后端架构设计，风险较高
**影响**: v1.x 保持"离线优先"定位

## 如何使用本文档

### 👨‍💻 开发者

**在开始新特性前**:
1. 查看 [特性矩阵](#特性矩阵) → 确认该特性属于哪个版本
2. 找到对应版本章节 → 阅读详细描述
3. 检查"对应 Issue" → 链接到具体任务

**在规划 Sprint 时**:
1. 查看当前版本的"核心特性"表格
2. 按优先级（P0 → P1 → P2）选择任务
3. 估算工作量，分配到 Sprint

### 📋 产品经理

**在评审需求时**:
1. 新需求来了 → 在 [特性矩阵](#特性矩阵) 查找是否已规划
2. 如未规划 → 决定加入哪个版本（考虑优先级和依赖）
3. 更新对应版本的"新增特性"表格

**在向外部沟通时**:
1. 查看 [时间线概览](#时间线概览) → 获取大图
2. 引用具体版本的"主题"和"目标日期"

### 🧪 测试人员

**在编写测试计划时**:
1. 查看版本的"成功标准" → 这是验收测试的基础
2. 查看"改进特性" → 回归测试范围
3. 查看"风险" → 重点测试场景

### 👥 用户/利益相关者

**了解产品计划**:
- 快速浏览 [时间线概览](#时间线概览)
- 查看 [特性矩阵](#特性矩阵) 了解功能演进
- 阅读感兴趣版本的"主题"和"核心特性"

## 维护指南

### 更新频率

| 情况 | 操作 |
|------|------|
| **每个 Sprint 开始前** | 检查当前版本进度，更新 Status |
| **发现重大技术风险** | 更新"风险"部分，可能调整特性 |
| **用户反馈影响优先级** | 调整特性优先级（P0/P1/P2） |
| **版本发布后** | 更新 Status 为 ✅ Released，归档到 CHANGELOG |

### 修改规则

1. **新增特性**: 先确定版本 → 添加到"新增特性"表格 → 更新特性矩阵
2. **延后特性**: 在原版本标记为"延后" → 移动到新版本 → 记录到"决策记录"
3. **取消特性**: 从表格删除 → 记录到"决策记录"并说明理由

### 与 GitHub 集成

每个特性应该有对应的 GitHub Issue/Project:

```markdown
| 功能 | 描述 | 对应 Issue |
|------|------|-----------|
| 多语言支持 | 中文/英文/日文界面 | [#42](https://github.com/yourorg/u-safe/issues/42) |
```
