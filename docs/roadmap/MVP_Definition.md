# U-Safe MVP 定义

> 明确第一个可交付版本的功能范围、技术约束和成功标准。

**版本**: 1.0
**日期**: 2026-03-14
**状态**: Draft

---

## 1. MVP 目标

交付一个**可独立使用的 U 盘文件加密管理工具**，核心价值：

1. **军事级文件加密** — AES-256-GCM 认证加密，保护敏感文件
2. **非侵入式标签整理** — 虚拟标签系统，不改变物理路径
3. **跨平台兼容** — Windows 10/11 和 macOS (Intel/Apple Silicon)

## 3. 技术约束

### 3.1 架构约束

| 约束 | 说明 |
|------|------|
| **单用户模式** | 不需要用户管理/注册/登录系统 |
| **本地 SQLite** | 不需要网络数据库, 数据库文件存储在 U 盘 |
| **单 U 盘** | 不需要多设备同步 |
| **离线优先** | 所有功能无需网络即可使用 |
| **无管理员权限** | 完全用户态运行, 免提权 |

### 3.2 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| 桌面框架 | Tauri | 2.0 |
| 后端 | Rust | latest stable |
| 前端 | React + TypeScript | 18.x |
| 样式 | TailwindCSS | 3.x |
| 数据库 | SQLite (via rusqlite) | 3.x |
| 加密 | AES-256-GCM | ring crate |
| KDF | Argon2id | argon2 crate |

### 3.3 数据库 Schema

MVP 使用 5 个核心表 (详见 `docs/spec/Database_Schema.md`):

1. **files** — 文件元数据 (路径、大小、加密状态、哈希)
2. **tags** — 层级标签 (名称、父标签、层级、完整路径)
3. **file_tags** — 文件-标签多对多关联
4. **encryption_meta** — 加密元数据 (salt, nonce, MAC)
5. **config** — 键值配置存储

### 3.4 加密参数

| 参数 | 值 |
|------|-----|
| 算法 | AES-256-GCM (AEAD) |
| 密钥长度 | 256 bits |
| 分块大小 | 64 KB |
| KDF | Argon2id (mem=64MB, time=3, lanes=1) |
| Salt | 32 bytes 随机 |
| Nonce | 12 bytes/块 随机 |

## 4. 成功标准

### 4.1 功能验收

| # | 标准 | 验证方法 |
|---|------|----------|
| S1 | 能加密/解密至少 100MB 的单个文件 | 手动测试: 选择 100MB 文件, 执行加密 → 解密, 验证内容一致 |
| S2 | 能管理至少 100 个标签 | 手动测试: 创建 100 个标签 (含多级层级), 验证查询性能 |
| S3 | 能在 Windows 10/11 上正常运行 | 在 Windows 环境编译并运行完整流程 |
| S4 | 能在 macOS (Intel + Apple Silicon) 上正常运行 | 在 macOS 环境编译并运行完整流程 |
| S5 | 冷启动时间 < 2 秒 (SSD U盘) | 使用 SSD U 盘测试冷启动, 计时到主界面渲染完成 |
| S6 | 应用体积 < 10 MB | 检查 release build 的二进制大小 |

### 4.2 安全验收

| # | 标准 | 验证方法 |
|---|------|----------|
| S7 | 密码错误时无法解密文件 | 输入错误密码, 验证解密失败并提示错误 |
| S8 | 加密文件无法被直接读取 | 用文本编辑器打开加密文件, 验证内容不可读 |
| S9 | 解密临时文件关闭后被删除 | 解密打开文件, 关闭后检查临时目录, 验证文件已删除 |
| S10 | 密码不在日志或内存 dump 中出现 | 代码审查: 验证使用 zeroize 清零敏感内存 |

### 4.3 用户体验验收

| # | 标准 | 验证方法 |
|---|------|----------|
| S11 | 加密/解密过程有进度反馈 | 加密大文件时观察进度条更新 |
| S12 | 标签操作响应时间 < 200ms | 创建/编辑/删除标签, 感知延迟 |
| S13 | 视图切换无明显卡顿 | 在 100+ 文件场景下切换物理/标签视图 |

## 5. 实施优先级

### P0 — 核心闭环 (必须先完成)

没有这些功能, 产品无法使用。

```
密码设置/验证 → 文件加密/解密 → 基础文件列表
```

**功能**: F1, F2, F3, F4, F9
**交付物**: 能加密和解密文件的最小可用版本

### P1 — 标签系统 (核心差异化)

标签管理是产品的差异化特性。

```
标签创建 → 文件打标签 → 标签视图 → 视图切换
```

**功能**: F5, F8, F10, F11
**交付物**: 完整的标签管理 + 双视图切换

### P2 — 完善体验

提升易用性的辅助功能。

```
标签编辑/删除 + 文件操作 + 进度显示
```

**功能**: F6, F7, F12, F13, F14, F15
**交付物**: 完整的文件和标签管理操作

## 7. 存储布局

MVP 采用以下 U 盘存储结构:

```
U盘根目录/
├── U-Safe.exe              # Windows 启动入口
├── U-Safe.app/             # macOS 启动入口
└── .u-safe/                # 系统隐藏目录
    ├── metadata.db         # SQLite 数据库 (标签、文件元数据)
    ├── data/               # 加密后的文件数据
    │   └── {file_id}.enc   # 加密分块数据
    └── keys/               # 密钥存储
        └── master.key.enc  # 加密的主密钥
```

## 8. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 加密性能不足 | 大文件加密过慢 | 使用 AES-NI 硬件加速, 异步 I/O |
| 跨平台兼容 | macOS/Windows 行为差异 | 早期进行跨平台编译测试 (M1) |
| SQLite 并发 | 多线程写入冲突 | 单用户模式, WAL 模式 |
| U 盘断电 | 加密中途拔出 U 盘导致数据损坏 | 原子写入 (临时文件 + rename), 事务保护 |
| 密码遗忘 | 无法恢复加密文件 | MVP 不提供恢复机制, 首次设置时明确告知用户 |

## 10. 决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| MVP 语言 | 仅中文 | 目标用户为中文用户, 多语言增加复杂度 |
| 搜索功能 | 不包含 | FTS5 可后续添加, MVP 用标签过滤足够 |
| 批量操作 | 不包含 | 单文件操作已满足基本需求 |
| U 盘拔出检测 | 不包含 | 需要平台特定实现, 用原子写入保护数据完整性 |
| 密码恢复 | 不提供 | 安全考虑, 避免增加攻击面 |
