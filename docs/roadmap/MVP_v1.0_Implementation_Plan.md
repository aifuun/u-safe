## 🎯 总览

### 核心价值主张

交付一个**可独立使用的 U 盘文件加密管理工具**，实现三大核心价值：

1. **军事级文件加密** — AES-256-GCM 认证加密，保护敏感文件
2. **非侵入式标签整理** — 虚拟标签系统，不改变物理路径
3. **跨平台兼容** — Windows 10/11 和 macOS (Intel/Apple Silicon)

### 成功标准（13条）

**功能验收** (6 条):
- ✅ 能加密/解密 100MB 文件 (性能 > 50 MB/s)
- ✅ 能管理 100 个层级标签
- ✅ Windows 10/11 正常运行
- ✅ macOS (Intel + Apple Silicon) 正常运行
- ✅ 冷启动 < 2 秒，应用体积 < 10 MB
- ✅ 零 P0 级别 Bug

**安全验收** (4 条):
- ✅ 错误密码无法解密
- ✅ 加密文件无法直接读取
- ✅ 临时文件自动删除
- ✅ 密码不在日志/内存中

**用户体验验收** (3 条):
- ✅ 加密/解密有进度反馈
- ✅ 标签操作 < 200ms
- ✅ 视图切换无卡顿

## ⚠️ 风险评估

### 高风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| **加密性能不足** | 大文件加密过慢，用户体验差 | 中 | • 使用 AES-NI 硬件加速<br>• 异步 I/O 避免阻塞<br>• 在 M2 阶段性能测试 |
| **跨平台兼容问题** | macOS/Windows 行为差异 | 高 | • M1 阶段立即进行跨平台编译<br>• 使用 Tauri 官方测试工具<br>• 准备双系统开发环境 |

### 中风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| **SQLite 并发冲突** | 多线程写入失败 | 低 | • 单用户模式设计<br>• 启用 WAL 模式<br>• 使用连接池管理 |
| **U盘断电数据损坏** | 加密中途拔出导致损坏 | 中 | • 原子写入 (临时文件 + rename)<br>• 事务保护<br>• M2 测试断电场景 |

### 低风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| **密码遗忘** | 用户无法恢复文件 | 低 | • 首次设置时明确告知<br>• 不提供恢复机制（安全考虑） |

## 🏗️ 架构设计

### 技术栈

```
┌─────────────────────────────────────────┐
│         前端 (React 18 + TS)            │
│  • UI 组件 (TailwindCSS)                │
│  • 状态管理 (React Hooks)               │
│  • IPC 调用 (@tauri-apps/api)           │
└─────────────┬───────────────────────────┘
              │ Tauri IPC
┌─────────────▼───────────────────────────┐
│         后端 (Rust)                     │
│  • 加密引擎 (ring + argon2)             │
│  • 数据库操作 (rusqlite)                │
│  • 文件 I/O (std::fs + tokio)           │
└─────────────┬───────────────────────────┘
              │
┌─────────────▼───────────────────────────┐
│      U 盘存储 (.u-safe/)                │
│  • metadata.db (SQLite)                 │
│  • data/{file_id}.enc (加密文件)        │
│  • keys/master.key.enc (主密钥)         │
└─────────────────────────────────────────┘
```

### 数据库 Schema (5 张表)

```sql
-- 1. files: 文件元数据
CREATE TABLE files (
  id TEXT PRIMARY KEY,
  path TEXT NOT NULL,
  size INTEGER NOT NULL,
  encrypted BOOLEAN NOT NULL DEFAULT 0,
  hash TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- 2. tags: 层级标签
CREATE TABLE tags (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  parent_id TEXT REFERENCES tags(id),
  level INTEGER NOT NULL,
  full_path TEXT NOT NULL,
  created_at TEXT NOT NULL
);

-- 3. file_tags: 文件-标签多对多关联
CREATE TABLE file_tags (
  file_id TEXT REFERENCES files(id),
  tag_id TEXT REFERENCES tags(id),
  created_at TEXT NOT NULL,
  PRIMARY KEY (file_id, tag_id)
);

-- 4. encryption_meta: 加密元数据
CREATE TABLE encryption_meta (
  file_id TEXT PRIMARY KEY REFERENCES files(id),
  salt BLOB NOT NULL,
  nonce BLOB NOT NULL,
  mac BLOB NOT NULL
);

-- 5. config: 键值配置
CREATE TABLE config (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
```

### 加密参数

```yaml
Algorithm: AES-256-GCM (AEAD)
Key Length: 256 bits
Block Size: 64 KB
KDF: Argon2id
  - Memory: 64 MB
  - Iterations: 3
  - Lanes: 1
Salt: 32 bytes (随机)
Nonce: 12 bytes/块 (随机)
```

### 存储布局

```
U盘根目录/
├── U-Safe.exe              # Windows 启动入口
├── U-Safe.app/             # macOS 启动入口
└── .u-safe/                # 系统隐藏目录
    ├── metadata.db         # SQLite 数据库
    ├── data/               # 加密文件存储
    │   └── {file_id}.enc
    └── keys/               # 密钥存储
        └── master.key.enc  # 加密的主密钥
```

## 🎯 成功标准详细

### 功能验收 (6 条)

- [ ] **S1**: 能加密/解密至少 100MB 的单个文件
  - **验证**: 手动测试，选择 100MB 文件，执行加密 → 解密，验证内容一致
  - **性能**: 加密速度 > 50 MB/s

- [ ] **S2**: 能管理至少 100 个标签
  - **验证**: 创建 100 个标签（含多级层级），验证查询性能
  - **性能**: 标签操作响应 < 200ms

- [ ] **S3**: 能在 Windows 10/11 上正常运行
  - **验证**: 在 Windows 环境编译并运行完整流程

- [ ] **S4**: 能在 macOS (Intel + Apple Silicon) 上正常运行
  - **验证**: 在 macOS 环境编译并运行完整流程

- [ ] **S5**: 冷启动时间 < 2 秒 (SSD U盘)
  - **验证**: 使用 SSD U 盘测试冷启动，计时到主界面渲染完成

- [ ] **S6**: 应用体积 < 10 MB
  - **验证**: 检查 release build 的二进制大小

### 安全验收 (4 条)

- [ ] **S7**: 密码错误时无法解密文件
  - **验证**: 输入错误密码，验证解密失败并提示错误

- [ ] **S8**: 加密文件无法被直接读取
  - **验证**: 用文本编辑器打开加密文件，验证内容不可读

- [ ] **S9**: 解密临时文件关闭后被删除
  - **验证**: 解密打开文件，关闭后检查临时目录，验证文件已删除

- [ ] **S10**: 密码不在日志或内存 dump 中出现
  - **验证**: 代码审查，验证使用 zeroize 清零敏感内存

### 用户体验验收 (3 条)

- [ ] **S11**: 加密/解密过程有进度反馈
  - **验证**: 加密大文件时观察进度条更新

- [ ] **S12**: 标签操作响应时间 < 200ms
  - **验证**: 创建/编辑/删除标签，感知延迟

- [ ] **S13**: 视图切换无明显卡顿
  - **验证**: 在 100+ 文件场景下切换物理/标签视图
