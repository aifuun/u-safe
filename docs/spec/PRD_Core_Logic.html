1. 加密/解密流程

1.1 用户首次设置密码流程

用户首次启动 U-Safe，需要设置主密码。

sequenceDiagram participant User as 用户 participant UI as 前端 UI participant Rust as Rust 后端 participant DB as SQLite User->>UI: 启动 U-Safe UI->>Rust: 检查是否已初始化 Rust->>DB: 查询 config 表 alt 未初始化 DB-->>Rust: NULL or false Rust-->>UI: 返回"未初始化" UI->>User: 显示"设置主密码"界面 User->>UI: 输入密码 + 确认密码 UI->>Rust: initialize_master_password(password) Rust->>Rust: 生成 32 bytes Salt Rust->>Rust: Argon2id 派生密钥 (64MB, 3 iterations) Rust->>Rust: 加密主密钥 Rust->>DB: 存储配置 Rust-->>UI: 返回成功 UI->>User: 显示"密码设置成功" else 已初始化 DB-->>Rust: true Rust-->>UI: 返回"需要解锁" UI->>User: 显示"输入密码解锁" end

关键步骤

  1. 密码强度验证: 最短 12 字符，包含大小写、数字、符号（使用 zxcvbn 库）
  2. Argon2id 配置: 64MB 内存 + 3 次迭代 + 单线程
  3. 主密钥生成: 256 bits 随机密钥，使用派生密钥加密
  4. 存储: Salt 存入 config 表，加密主密钥存入 .usafe/keys/master.key.enc
// Argon2id 密钥派生（基于 ADR 0003）
let config = argon2::Config {
    variant: argon2::Variant::Argon2id,
    version: argon2::Version::Version13,
    mem_cost: 65536,      // 64 MB
    time_cost: 3,         // 3 iterations
    lanes: 1,             // single-threaded
    thread_mode: argon2::ThreadMode::Sequential,
};

let salt = generate_random_bytes(32);  // 32 bytes
let derived_key = argon2::hash_raw(
    password.as_bytes(),
    &salt,
    &config
)?;

1.2 文件加密流程

用户在 U-Safe 中新增文件或标记现有文件为"加密"。

flowchart TD A[用户选择文件加密] --> B{文件是否已在数据库} B -->|否| C[添加文件元数据到 files 表] B -->|是| D[检查 is_encrypted 状态] C --> E[生成文件 UUID] D -->|已加密| F[提示文件已加密] D -->|未加密| G[开始加密流程] E --> G G --> H[计算原始文件 SHA-256] H --> I[生成加密元数据] I --> J[Argon2id 派生文件密钥] J --> K[分块加密 64KB/chunk] K --> L[写入临时文件 .tmp.timestamp] L --> M[每块写入后 fsync] M --> N{所有块完成?} N -->|否| K N -->|是| O[原子重命名 temp → final] O --> P[更新 files 表 is_encrypted=1] P --> Q[插入 encryption_meta] Q --> R[删除原始文件] R --> S[加密完成] F --> T[结束] S --> T

⚡ 性能参数（基于 ADR 0003）

分块大小65,536 bytes (64 KB)
Nonce12 bytes per chunk（随机）
MAC16 bytes per chunk（GCM 自动）
Salt32 bytes（每文件独立）
// 64KB 分块加密
let cipher = Aes256Gcm::new(&file_key);
let temp_file = format!("{}.tmp.{}", output_path, timestamp());

for chunk_index in 0..total_chunks {
    // 1. 读取 64KB 数据
    let chunk_data = read_chunk(&file_path, chunk_index, 65536)?;

    // 2. 生成 Nonce (12 bytes)
    let nonce = generate_random_bytes(12);
    nonce_list.extend_from_slice(&nonce);

    // 3. AES-256-GCM 加密
    let ciphertext = cipher.encrypt(&nonce, chunk_data.as_ref())?;

    // 4. 提取 MAC (最后 16 bytes)
    let mac = &ciphertext[ciphertext.len() - 16..];
    mac_list.extend_from_slice(mac);

    // 5. 写入 + fsync
    write_chunk(&temp_file, &ciphertext)?;
    fsync(&temp_file)?;
}

// 原子重命名
std::fs::rename(&temp_file, &final_path)?;

1.3 文件解密流程

flowchart TD A[用户打开加密文件] --> B[验证主密码] B -->|失败| C[提示密码错误] B -->|成功| D[查询 encryption_meta] D --> E[Argon2id 派生文件密钥] E --> F[分块解密] F --> G[验证 MAC] G -->|失败| H[提示文件损坏或被篡改] G -->|成功| I[解密数据块] I --> J{所有块完成?} J -->|否| F J -->|是| K[合并数据到内存] K --> L[调用系统默认应用打开临时文件] L --> M[用户关闭应用] M --> N[立即删除临时文件] N --> O[清零内存缓冲区] C --> P[结束] H --> P O --> P

🔒 安全考虑

  • 解密数据仅存在于内存和临时文件（用户关闭立即删除）
  • 临时文件设置严格权限（仅当前用户可读 0o600）
  • 内存数据使用后立即清零（zeroize crate）
  • 不在宿主电脑硬盘留下明文缓存

1.4 密码验证机制

sequenceDiagram participant User as 用户 participant UI as 前端 participant Rust as Rust 后端 participant Memory as 内存缓存 User->>UI: 输入密码 UI->>Rust: verify_password(password) Rust->>Rust: 读取 Salt Rust->>Rust: Argon2id 派生密钥 (~500ms) Rust->>Rust: 尝试解密主密钥 alt 解密成功 Rust->>Memory: 缓存主密钥 (TTL: 15分钟) Rust-->>UI: 返回成功 UI->>User: 显示主界面 else 解密失败 Rust-->>UI: 返回"密码错误" UI->>User: 提示重新输入 end Note over Memory: 15分钟后自动清除 Note over Rust: U盘拔出立即清除

✅ 密钥缓存策略

  • • TTL: 15 分钟
  • • 过期自动清除
  • • U盘拔出立即清零
  • • 内存加锁（mlock）

⚠️ 重试限制

  • • 最多 5 次失败尝试
  • • 连续失败锁定 5 分钟
  • • 成功后重置计数器

2.2 标签编辑和删除

编辑标签名称时，需要更新 full_path 和所有子标签的路径。删除标签会级联删除所有子标签（ON DELETE CASCADE）。

2.3 标签批量操作

// 批量打标签（事务保证）
fn batch_tag_files(file_ids: &[String], tag_id: &str) -> Result<()> {
    let tx = db.transaction()?;

    for file_id in file_ids {
        tx.execute(
            "INSERT OR IGNORE INTO file_tags (file_id, tag_id, tagged_at)
             VALUES (?, ?, ?)",
            params![file_id, tag_id, Utc::now().to_rfc3339()],
        )?;
    }

    // 更新标签使用次数
    tx.execute(
        "UPDATE tags SET usage_count = usage_count + ? WHERE tag_id = ?",
        params![file_ids.len(), tag_id],
    )?;

    tx.commit()
}

    UNION ALL

📝 实现细节

fn add_file(source_path: &Path, encrypt: bool, tags: &[String]) -> Result<()> {
    // 1. 复制或加密文件
    let file_id = Uuid::new_v4().to_string();
    let dest_path = if encrypt {
        encrypt_file_to_usb(source_path, &file_id)?
    } else {
        copy_file_to_usb(source_path, &file_id)?
    };

    // 2. 添加元数据
    insert_file_metadata(&file_id, &dest_path, encrypt)?;

    // 3. 打标签
    for tag_id in tags {
        insert_file_tag(&file_id, tag_id)?;
    }

    Ok(())
}

5. 错误处理和边界情况

🔌 U盘意外拔出

  • ✅ 加密文件原子写入（临时文件 + rename）
  • ✅ 数据库 WAL 模式（Write-Ahead Logging）
  • ✅ 每次写入后 fsync
  • ✅ 内存密钥立即清零

🔑 密码忘记

❌ 不提供"忘记密码"功能（安全设计）

✅ 可选备份：导出主密钥为 BIP39 恢复词

💥 文件损坏检测

启动时扫描所有加密文件的 MAC，标记损坏文件
