# 密码重置指南

## 概述

如果您忘记了 U-Safe 的主密码，可以使用密码重置功能重新设置密码。

**⚠️ 重要警告：**
- 重置密码将**永久删除**所有加密文件的访问权限
- 已加密的文件将**无法解密**
- 所有标签和文件记录将被清空
- **此操作不可撤销**

## 何时使用密码重置

只有在以下情况下才应使用密码重置功能：
- ✅ 您确实忘记了主密码
- ✅ 您已尝试所有可能记得的密码
- ✅ 您接受所有加密文件将永久丢失
- ✅ 您愿意从头开始使用 U-Safe

## 重置流程

### 步骤 1: 进入重置流程

1. 在登录页面，找到「忘记密码？」提示
2. 点击「重置密码并清空数据」按钮
3. 进入警告页面

### 步骤 2: 查看警告信息

警告页面会显示将要删除的数据统计：

- 🔒 **加密文件数量**：已加密的文件个数
- 📄 **所有文件记录**：数据库中的所有文件记录
- 🏷️ **标签数量**：您创建的所有标签
- 💾 **数据库大小**：当前数据库占用空间

**重要说明：**
- 已加密的文件将永久无法解密
- 所有标签和文件记录将被清空
- 数据库将备份到 `metadata.db.backup`
- 您需要重新设置主密码

**操作选项：**
- 点击「取消」返回登录页面（不执行重置）
- 点击「我了解风险，继续」进入确认页面

### 步骤 3: 最终确认

确认页面包含多重安全检查：

1. **勾选确认复选框（必须）：**
   - [ ] 我理解并接受所有加密文件将**永久丢失**，无法恢复
   - [ ] 我已尝试所有可能的密码，确认**无法找回**

2. **输入确认文本（必须）：**
   - 在文本框中输入 `DELETE`（必须大写）
   - 这是为了防止误操作

3. **点击「确认重置并清空数据」按钮**
   - 只有完成以上两步，按钮才会启用

### 步骤 4: 重置执行

点击确认后，系统将：

1. ✅ 归档数据库到 `metadata.db.backup`
2. ✅ 删除密码哈希文件
3. ✅ 清空所有数据表
4. ✅ 跳转到密码设置页面

整个过程通常在几秒内完成。

### 步骤 5: 重新设置密码

重置完成后，您将看到密码设置页面：

1. 输入新的主密码
2. 确认密码
3. 点击「设置密码」

现在您可以使用新密码重新开始使用 U-Safe。

## 数据恢复（高级用户）

虽然密码重置会清空数据库，但系统会创建备份文件。如果您是高级用户，可以尝试手动恢复：

### 备份文件位置

**macOS:**
```
~/Library/Application Support/.u-safe/metadata.db.backup
```

**Windows:**
```
%APPDATA%\.u-safe\metadata.db.backup
```

**Linux:**
```
~/.local/share/.u-safe/metadata.db.backup
```

### 恢复步骤（需技术知识）

1. 退出 U-Safe 应用
2. 将 `metadata.db.backup` 重命名为 `metadata.db`
3. 恢复密码哈希文件（如果您有备份）
4. 重新启动应用

**注意：** 如果没有密码哈希文件，即使恢复数据库也无法解密文件。

## 常见问题

### Q: 重置后能恢复数据吗？

A: **不能**。已加密的文件需要原始密码才能解密。重置密码后，原始密钥永久丢失，加密文件无法恢复。

### Q: 我可以只重置密码而不删除数据吗？

A: **不可以**。密码与加密密钥绑定，更改密码意味着无法解密旧文件，因此必须清空数据。

### Q: 重置会删除 U 盘上的原始文件吗？

A: **不会**。重置只影响 U-Safe 数据库和加密文件记录。您 U 盘上的原始文件（如果未被加密覆盖）仍然存在。

### Q: 我还有其他选择吗？

A: 如果您还记得部分密码，建议：
1. 尝试所有可能的密码组合
2. 检查密码管理器或笔记
3. 尝试常用的密码变体

只有在完全确认无法找回密码时才使用重置功能。

### Q: 重置需要多长时间？

A: 通常在 5-10 秒内完成，具体取决于数据库大小。

### Q: 重置过程中出错怎么办？

A: 如果重置失败：
1. 查看错误提示信息
2. 检查磁盘空间是否充足
3. 重启应用后重试
4. 如果仍然失败，联系技术支持

## 预防措施

为避免需要重置密码：

1. **使用密码管理器**：将主密码保存在密码管理器中（如 1Password, LastPass）
2. **写下密码**：在安全的地方写下密码（物理存储）
3. **定期登录**：经常使用 U-Safe 以保持密码记忆
4. **使用助记词**：选择容易记住但难以猜测的密码

## 安全说明

密码重置功能设计了多重确认机制：
- ✅ 警告页面显示数据统计
- ✅ 确认页面的两个复选框
- ✅ 输入 "DELETE" 文本确认
- ✅ 数据库自动备份

这些措施确保您不会意外重置密码。

## 技术细节

如果您对技术细节感兴趣：

- **加密算法**：AES-256-GCM

- **备份格式**：SQLite 数据库文件
