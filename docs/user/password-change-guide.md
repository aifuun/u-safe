# 修改密码指南

## 概述

U-Safe 支持在登录后修改主密码，而无需重新加密所有文件。得益于 Master Key Wrapping 架构，密码修改操作可在 <1 秒内完成，所有已加密文件保持可用。

**✅ 优势：**
- 快速完成（<1 秒，与文件数量无关）
- 所有加密文件保持可用
- 标签和文件记录不受影响
- 操作安全且原子性（失败自动回滚）

**⚠️ 重要提示：**
- 修改密码后，请务必记住新密码
- 建议使用密码管理器（如 1Password、Bitwarden）保存密码
- 忘记新密码将无法恢复加密文件

## 何时使用密码修改

适合在以下情况使用密码修改功能：
- ✅ 您记得当前密码
- ✅ 需要更换更强的密码
- ✅ 密码可能已泄露，需要更换
- ✅ 定期安全维护（建议每 3-6 个月更换一次）

**不适合的情况：**
- ❌ 忘记当前密码 → 请使用[密码重置](./password-reset-guide.md)功能

## 修改流程

### 步骤 1: 进入设置页面

1. 登录 U-Safe 应用
2. 在文件管理页面，点击顶部的「⚙️ 设置」按钮
3. 进入设置主页面

### 步骤 2: 选择修改密码

1. 在设置页面，找到「安全」分类
2. 点击「🔑 修改主密码」选项
3. 进入密码修改页面

### 步骤 3: 填写密码信息

密码修改页面包含三个密码输入框：

1. **当前密码**
   - 输入您现在正在使用的主密码
   - 用于验证您的身份
   - 如果输入错误，系统会提示「旧密码错误」

2. **新密码**
   - 输入您想要设置的新密码
   - 密码要求：
     - 至少 8 个字符
     - 包含大写字母 (A-Z)
     - 包含小写字母 (a-z)
     - 包含数字 (0-9)
   - 系统会实时显示密码强度提示

3. **确认新密码**
   - 再次输入新密码
   - 确保两次输入一致
   - 如果不一致，系统会提示「两次输入的新密码不一致」

### 步骤 4: 提交修改

1. 确认所有信息填写正确
2. 点击「确认修改」按钮
3. 系统将执行以下操作（<1 秒）：
   - ✅ 验证当前密码
   - ✅ 使用新密码重新加密主密钥
   - ✅ 更新密码哈希文件
   - ✅ 确保操作原子性（失败自动回滚）

4. 修改成功后，系统会显示提示：「密码修改成功！请使用新密码重新登录。」
5. 自动返回设置页面

### 步骤 5: 使用新密码登录

首次修改密码后：
1. 返回登录页面
2. 使用新密码登录
3. 验证密码修改成功

## 密码要求详解

### 最低要求
- 最少 8 个字符
- 包含至少一个大写字母
- 包含至少一个小写字母
- 包含至少一个数字

### 推荐做法
- 使用 12-16 个字符（更安全）
- 混合使用字母、数字、特殊字符
- 避免使用常见密码（如 "Password123"）
- 不要使用个人信息（生日、姓名等）
- 每个服务使用不同的密码

### 密码强度示例

**弱密码（不推荐）：**
- ❌ `12345678`（纯数字）
- ❌ `password`（纯小写）
- ❌ `Password`（缺少数字）
- ❌ `Pass123`（太短）

**强密码（推荐）：**
- ✅ `MyS3cure!Pass`（大小写 + 数字 + 特殊字符）
- ✅ `Tr0ub4dor&3`（混合字符类型）
- ✅ `C0rrect-H0rse-Battery`（长且易记）

## 错误处理

### 常见错误及解决方法

#### 1. 「旧密码错误」
- **原因**：输入的当前密码不正确
- **解决方法**：
  1. 确认大小写是否正确
  2. 检查是否有多余空格
  3. 尝试复制粘贴密码（避免输入错误）
  4. 如果确实忘记密码，使用[密码重置](./password-reset-guide.md)功能

#### 2. 「新密码不能与旧密码相同」
- **原因**：新密码与当前密码一致
- **解决方法**：输入一个不同的新密码

#### 3. 「两次输入的新密码不一致」
- **原因**：新密码和确认密码输入不一致
- **解决方法**：
  1. 点击密码输入框旁的「👁️」图标显示密码
  2. 确认两次输入完全相同

#### 4. 「密码长度至少 8 个字符」
- **原因**：新密码太短
- **解决方法**：输入至少 8 个字符的密码

#### 5. 「密码需要包含大小写字母和数字」
- **原因**：新密码不符合强度要求
- **解决方法**：确保密码同时包含：
  - 大写字母（A-Z）
  - 小写字母（a-z）
  - 数字（0-9）

#### 6. 「密码修改失败，请重试」
- **原因**：系统错误或文件权限问题
- **解决方法**：
  1. 检查磁盘空间是否充足
  2. 确认应用有文件读写权限
  3. 重启应用后重试
  4. 查看应用日志获取详细错误信息

## 安全最佳实践

### 1. 使用密码管理器
强烈建议使用密码管理器保存主密码：
- **1Password**：跨平台支持，安全性高
- **Bitwarden**：开源免费，功能完善
- **LastPass**：免费版支持单设备

### 2. 定期更换密码
- 建议每 3-6 个月更换一次
- 如果怀疑密码泄露，立即更换
- 使用不同的密码（不要循环使用旧密码）

### 3. 密码备份
- 将密码写在安全的地方（物理存储）
- 告诉信任的人密码位置（紧急情况）
- 不要通过邮件或聊天软件发送密码

### 4. 密码强度
- 优先使用长密码（12+ 字符）胜过复杂短密码
- 使用助记词或短句（如 `Tr0ub4dor&3`）
- 避免使用键盘模式（如 `qwerty123`）

## 技术细节

### Master Key Wrapping 架构

U-Safe 使用双层密钥架构实现快速密码修改：

```
用户密码（可更改）
  ↓ Argon2id (KDF)
密码派生密钥 (KEK)
  ↓ AES-256-GCM 加密
主密钥（固定，32字节随机）
  ↓ AES-256-GCM
加密文件（永不改变）
```

**修改密码时发生了什么：**
1. 验证旧密码 → 解密主密钥（32 字节）
2. 用新密码派生新 KEK
3. 用新 KEK 重新加密主密钥（仅 32 字节）
4. 原子更新 `master.key` 和 `password.hash` 文件
5. 文件层加密不变（继续使用同一个主密钥）

**为什么快（<1 秒）：**
- 仅重新加密主密钥（32 字节），不重新加密文件
- 文件数量再多也不影响速度
- 操作原子性（失败自动回滚，不会损坏数据）

**文件位置：**
```
macOS:
~/Library/Application Support/.u-safe/keys/
  ├── password.hash  (Argon2id 哈希，验证密码)
  └── master.key     (加密后的主密钥)

Windows:
%APPDATA%\.u-safe\keys\
  ├── password.hash
  └── master.key

Linux:
~/.local/share/.u-safe/keys/
  ├── password.hash
  └── master.key
```

## 常见问题

### Q: 修改密码需要多长时间？
A: 通常 <1 秒完成，与文件数量无关。即使您有 10,000 个加密文件，密码修改时间也不变。

### Q: 修改密码后，已加密文件还能用吗？
A: **能**。所有已加密文件保持可用，使用新密码即可正常解密。

### Q: 修改密码会重新加密所有文件吗？
A: **不会**。得益于 Master Key Wrapping 架构，只需重新加密主密钥（32 字节），文件层加密不变。

### Q: 修改密码失败会怎样？
A: 操作具有原子性，失败自动回滚。旧密码仍然可用，数据不会损坏。

### Q: 可以多次修改密码吗？
A: 可以。您可以随时修改密码，次数不限。

### Q: 修改密码后需要重新登录吗？
A: 建议首次修改后重新登录，验证新密码可用。后续使用新密码登录即可。

### Q: 忘记新密码怎么办？
A: 如果忘记新密码，只能使用[密码重置](./password-reset-guide.md)功能（会清空所有数据）。因此请务必记住新密码或保存在密码管理器中。

### Q: 密码修改和密码重置有什么区别？

| 功能 | 修改密码 | 重置密码 |
|------|---------|---------|
| **前提条件** | 记得当前密码 | 忘记密码 |
| **数据保留** | ✅ 所有数据保留 | ❌ 清空所有数据 |
| **文件可用** | ✅ 文件正常使用 | ❌ 文件永久丢失 |
| **耗时** | <1 秒 | ~5 秒 |
| **使用场景** | 定期安全维护 | 紧急恢复访问 |

### Q: 修改密码安全吗？
A: 非常安全。操作使用原子文件更新（临时文件 + rename），失败自动回滚。即使在操作过程中断电，也不会损坏数据。

## 故障排除

### 问题 1: 点击「确认修改」无响应
**可能原因：**
- 有密码字段未填写
- 新密码不符合强度要求
- 两次新密码输入不一致

**解决方法：**
1. 确认所有密码字段已填写
2. 查看页面上的错误提示
3. 点击密码输入框旁的「👁️」图标检查密码内容

### 问题 2: 提示「旧密码错误」但密码确实正确
**可能原因：**
- 密码输入包含多余空格
- 大小写错误
- 使用了不同的键盘布局

**解决方法：**
1. 清空密码字段重新输入
2. 使用密码管理器复制粘贴密码
3. 检查键盘的 Caps Lock 是否开启
4. 确认输入法为英文输入

### 问题 3: 修改成功但使用新密码无法登录
**可能原因：**
- 新密码记忆错误
- 输入时大小写错误

**解决方法：**
1. 仔细回忆新密码的大小写和特殊字符
2. 如果确实忘记，使用[密码重置](./password-reset-guide.md)功能

### 问题 4: 应用崩溃或关闭
**可能原因：**
- 系统资源不足
- 应用 bug

**解决方法：**
1. 重启应用
2. 重新登录并再次尝试修改密码
3. 查看应用日志（如果启用了调试模式）
4. 如果问题持续，报告 bug 到 GitHub Issues
